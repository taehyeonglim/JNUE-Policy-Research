# GitHub Pages 탭 구조 완료 점검

점검일: 2026-05-03

## 목표

기존 단일 페이지형 GitHub Pages 대시보드를 탭 구조로 나누고, 각 탭마다 더 충실한 내용을 배치한다.

## 완료 기준과 증거

| 요구사항 | 반영 위치 | 증거 |
|---|---|---|
| 단일 긴 페이지를 탭 구조로 전환 | `docs/index.html`, `scripts/generate_pages_site.py` | `data-tab-target` 버튼 7개, `tab-panel` 7개 생성 |
| 탭별 내용 분리 | 같은 파일 | `개요`, `전공·수요`, `모델·로드맵`, `학칙`, `부록`, `검증`, `산출물` 탭 구성 |
| 각 탭 내용 보강 | `scripts/generate_pages_site.py` | 로드맵 표, 리스크 대응표, 부록 패키지 표, 감사 결과 표, A1·A2 설명 추가 |
| 탭 직접 진입 | `docs/index.html` | `#appendix`, `#law` 해시 진입 시 해당 탭 활성화 확인 |
| 그래프 리사이즈 | `scripts/generate_pages_site.py` | 탭 전환 시 `window.dispatchEvent(new Event("resize"))`, `Plotly.Plots.resize` 호출 |
| Pages 산출물 재생성 | `docs/index.html` | `python3 scripts/generate_pages_site.py` 실행 완료 |

## 검증 명령

- `python3 scripts/generate_pages_site.py`
- `rg -n 'class="tab-nav"|role="tab"|class="tab-panel|data-tab-target' docs/index.html`
- `node --check /tmp/jnue_script_*.js`
- Chrome headless `--dump-dom http://127.0.0.1:8007/#appendix`
- Chrome headless `--dump-dom http://127.0.0.1:8007/#law`

## 검증 결과

- 탭 버튼 수: 7개.
- 탭 패널 수: 7개.
- JavaScript 문법 검사: 전체 inline script 통과.
- `#appendix`: 부록 탭 `aria-selected="true"`, `tab-appendix active` 확인.
- `#law`: 학칙 탭 `aria-selected="true"`, `tab-law active` 확인.
