# HWPX 다운로드 손상 수정 검증 (2026-05-03)

## 문제

GitHub Pages 산출물 탭의 `최종보고서 HWPX 다운로드` 파일은 ZIP 무결성 검사는 통과했지만, 내부 본문 XML이 `2016/HwpmlSection`, `2016/HwpmlParagraph` 기반의 임의 축약 구조로 생성되어 한컴에서 손상 파일로 판정될 수 있었다.

## 수정

- `scripts/generate_hwpx.py`를 정상 HWPX 템플릿 기반 생성 방식으로 교체했다.
- 한컴 생성 HWPX의 패키지 구조, `version.xml`, `settings.xml`, `Contents/header.xml`, `Contents/content.hpf`, `META-INF/*`를 보존하고 `Contents/section0.xml`과 `Preview/PrvText.txt`만 교체한다.
- 새 본문 XML은 `http://www.hancom.co.kr/hwpml/2011/section`, `http://www.hancom.co.kr/hwpml/2011/paragraph` 네임스페이스와 템플릿의 문단/글자 속성 참조를 사용한다.
- `scripts/validate_hwpx.py`를 추가해 Pages 배포 전 다운로드 파일의 HWPX 구조와 핵심 본문 포함 여부를 검사한다.
- `scripts/generate_pages_site.py`가 HWPX를 `docs/assets/`로 복사하기 전에 같은 검사를 수행하도록 했다.

## 결론

다운로드용 HWPX는 정상 HWPX 템플릿 기반 패키지로 재생성되었고, Pages 배포 산출물에도 동일 파일이 복사되도록 검증 장치를 추가했다.
