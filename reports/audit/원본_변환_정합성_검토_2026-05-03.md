# 원본 파일 변환 및 데이터 정합성 검토

검토일: 2026-05-03

후속 조치: 본 검토에서 미변환으로 분류한 HWP/HWPX 10건은 같은 날 모두 변환 완료했다. 최신 상태는 `reports/audit/문서형_원본_10건_MD_변환완료_검증_2026-05-03.md`를 기준으로 한다.

## 1. 검토 목적

`data/` 하위 원본 자료가 현재 보고서 작성에 사용할 수 있는 `.md`, `.txt` 또는 구조화 데이터 로더 형태로 정확히 반영되어 있는지 점검하였다. 문서형 원본은 변환물 존재 여부와 내용 보존성을 확인했고, 표 데이터는 `src/data_loader.py` 및 `scripts/audit_report_facts.py`가 원자료를 읽어 핵심 수치를 재현하는지 확인했다.

## 2. 결론

현재 상태에서 **모든 원본 문서가 `.md` 등으로 변환되어 있다고 볼 수 없다.**

- 변환 정합성이 확인된 문서: 전국교대 학칙/규정 8건, PDF 원본 6건.
- 직접 변환물이 없는 문서형 원본: HWP 5건, HWPX 5건.
- 스프레드시트 원자료는 `.md` 변환 대상이 아니라 로더가 직접 읽는 구조이며, 핵심 로더와 팩트 감사는 실행 통과했다.
- `FACT_LEDGER` 기준 핵심 수치 57건 중 PASS 53건, CHECK 4건이 남아 있다. CHECK 4건은 변환 오류라기보다 기준 선택 또는 자료 간 정의 차이에 관한 의사결정 항목이다.

따라서 현재 원고와 대시보드의 핵심 수치는 상당 부분 검증되어 있으나, **원본 변환 커버리지 관점에서는 미완료**다. 제출용 최종 검수 전에는 직접 변환물이 없는 HWP/HWPX 10건을 변환하거나, 보고서에서 참조하지 않는 보조 자료로 명시적으로 분류해야 한다.

## 9. 조치 권고

1. 직접 변환물이 없는 HWP/HWPX 10건을 `*.md` 또는 `_raw_text/*.txt`로 변환한다.
2. 변환 후 `audit_hwp_pdf_markdown.py`와 유사한 조문·표·숫자 토큰 대조 검사를 해당 파일군에도 확대한다.
3. 보고서 본문에서 HWP/HWPX 미변환 원본을 직접 인용한 부분이 있는지 역추적한다. 직접 인용이 있다면 최종본 전 변환 검증이 필수다.
4. `POLICY-2026-STD`와 `DNUE-INSTALL-YEAR`는 최종 신청서 작성 기준으로 확정하고, `FACT_LEDGER.csv`의 CHECK를 PASS 또는 의사결정 완료 상태로 정리한다.
5. `.env`는 현재 미추적 파일이므로 감사 산출물 커밋 시 포함하지 않는다.
