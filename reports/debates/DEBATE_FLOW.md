# 하이브리드 칼포퍼 토론 플로우 (Hybrid Karl Popper Debate)

> 버전: 2.0 (2026-04-09 개정)
> 이전 버전: 4라운드 순차 토론 → **8라운드 교차심문 포함 토론**으로 전환

---

## 설계 철학

칼 포퍼(Karl Popper)의 **반증가능성(falsifiability)** 원리에 기반한 토론 구조.
핵심은 **교차심문(Cross-Examination)**을 통해 각 논증의 약점을 직접 검증하는 것이다.

### 기존 4라운드 구조의 한계

| 문제 | 설명 |
|------|------|
| 허수아비 논증 | 반박가가 입안자의 실제 논거가 아닌 자의적 논거를 공격 |
| 핵심 쟁점 회피 | 양쪽이 서로 다른 차원에서 주장하여 진정한 충돌 부재 |
| 논증 깊이 부족 | "왜 그 데이터가 그 결론을 지지하는가?"를 파고들 기회 없음 |
| 일방적 재반박 | 입안자만 재반박, 반박가의 최종 반박 기회 부재 |

### 하이브리드 구조의 해결

- **교차심문**: 상대 논거를 직접 인용하여 질문으로 검증
- **양방향 반박**: 입안자·반박가 모두 최종 반박 기회 부여
- **분석가 기반**: 공유 팩트시트로 동일한 증거 기반에서 토론
- **조언가 판정**: 교차심문 승패까지 포함한 종합 평가

## 8라운드 토론 구조

```
┌─────────────────────────────────────────────────┐
│  R1  분석가 팩트시트        [공유 증거 기반 구축]  │
├─────────────────────────────────────────────────┤
│  R2  입안자 찬성 입론       [정책 당위성 논증]     │
├─────────────────────────────────────────────────┤
│  R3  반박가→입안자 교차심문  [찬성 논거 검증]      │
├─────────────────────────────────────────────────┤
│  R4  반박가 반대 입론       [예상 반대 논거 전개]   │
├─────────────────────────────────────────────────┤
│  R5  입안자→반박가 교차심문  [반대 논거 검증]      │
├─────────────────────────────────────────────────┤
│  R6  입안자 최종 반박       [교차심문 반영 종합]    │
├─────────────────────────────────────────────────┤
│  R7  반박가 최종 반박       [교차심문 반영 종합]    │
├─────────────────────────────────────────────────┤
│  R8  조언가 종합 판정       [6관점 + 교차심문 평가] │
└─────────────────────────────────────────────────┘
```

## 각 라운드 상세 규칙

### R1: 분석가 팩트시트

- **목적:** 토론의 공유 증거 기반(shared evidence base) 구축
- **에이전트:** 분석가 (Claude Opus)
- **규칙:**
  - 토론 주제와 관련된 모든 데이터를 원천 파일에서 직접 읽어 정리
  - 찬/반 어느 쪽에도 유리하게 선별하지 않음 (객관 중립)
  - 데이터 미확보 항목을 명시적으로 표기
  - 이후 모든 라운드의 논거는 이 팩트시트의 데이터만 인용 가능
    (팩트시트에 없는 데이터를 인용하려면 출처를 명시하고 분석가가 확인)

### R2: 입안자 찬성 입론 (Affirmative Constructive)

- **목적:** 정책 당위성의 체계적 논증
- **에이전트:** 정책입안자 (Claude Opus)
- **규칙:**
  - R1 팩트시트를 기반으로 3~5개 핵심 논거 전개
  - 각 논거는 반드시 [주장 → 근거 데이터 → 추론] 3단 구조
  - 논거에 번호를 부여 (교차심문에서 참조하기 위함)
  - 새로운 데이터 인용 시 출처 명시 필수

### R3: 반박가→입안자 교차심문 (Cross-Examination of Affirmative)

- **목적:** 찬성 논거의 약점을 질문으로 드러냄
- **에이전트:** 반박가가 질문, 입안자가 답변
- **교차심문 규칙:**
  1. **질문만 허용:** 반박가는 주장이 아닌 질문만 할 수 있음
  2. **논거 인용 필수:** 각 질문은 R2에서 입안자가 제시한 구체적 논거·데이터를 인용해야 함
  3. **질문 수 제한:** 핵심 질문 3~5개 (산만한 질문 남발 금지)
  4. **답변 형식:** 입안자는 [인정/부인/조건부 인정] + 근거로 답변
  5. **새 논거 금지:** 답변자는 새로운 논거를 펼칠 수 없고, 기존 논거를 방어만 가능
  6. **후속 질문 허용:** 답변이 불충분하면 1회 후속 질문 가능

- **교차심문 질문 유형:**
  | 유형 | 목적 | 예시 |
  |------|------|------|
  | 데이터 검증 | 인용 수치의 정확성·맥락 확인 | "서울교대 100% 충원은 수도권이라는 변수를 통제한 수치입니까?" |
  | 인과 추론 검증 | 상관관계를 인과로 비약하는지 확인 | "설치대학 +1.5%가 박사과정 때문이라는 근거는 무엇입니까?" |
  | 일반화 검증 | 특수 사례를 일반화하는지 확인 | "공주교대 100% 충원이 전주에도 적용된다는 근거는?" |
  | 전제 검증 | 숨겨진 전제를 드러냄 | "석사 미달의 원인이 박사과정 부재라는 전제의 근거는?" |
  | 반례 제시 | 논거를 무너뜨리는 반례 질문 | "경인교대는 설치 후 오히려 석사 경쟁률이 하락했는데, 이를 어떻게 설명합니까?" |

### R4: 반박가 반대 입론 (Negative Constructive)

- **목적:** 정책에 대한 체계적 반대 논거 전개
- **에이전트:** 반박가 (Codex GPT-5.4 xhigh)
- **규칙:**
  - R1 팩트시트 + R3 교차심문 결과를 기반으로 3~5개 반대 논거 전개
  - 교차심문에서 입안자가 인정한 약점을 적극 활용
  - 각 논거에 위험도(상/중/하) 부여
  - 논거에 번호를 부여 (R5 교차심문 참조용)

### R5: 입안자→반박가 교차심문 (Cross-Examination of Negative)

- **목적:** 반대 논거의 약점을 질문으로 드러냄
- **에이전트:** 입안자가 질문, 반박가가 답변
- **규칙:** R3과 동일한 교차심문 규칙 적용
  - 입안자는 R4의 반대 논거를 구체적으로 인용하여 질문
  - 반박가는 [인정/부인/조건부 인정] + 근거로 답변

### R6: 입안자 최종 반박 (Affirmative Rebuttal)

- **목적:** 전체 토론 결과를 반영한 찬성 측 최종 정리
- **에이전트:** 정책입안자 (Claude Opus)
- **규칙:**
  - R3 교차심문에서 드러난 자기 측 약점을 솔직히 인정하고 보완
  - R5 교차심문에서 드러난 반대 측 약점을 활용
  - **새로운 논거 제시 금지** — 기존 논거의 강화·보완만 가능
  - 최종 정리: "인정하는 한계 + 그럼에도 불구하고 + 핵심 주장 재확인"

### R7: 반박가 최종 반박 (Negative Rebuttal)

- **목적:** 전체 토론 결과를 반영한 반대 측 최종 정리
- **에이전트:** 반박가 (Codex GPT-5.4 xhigh)
- **규칙:**
  - R5 교차심문에서 드러난 자기 측 약점을 솔직히 인정
  - R3 교차심문에서 드러난 찬성 측 약점을 활용
  - **새로운 논거 제시 금지**
  - 최종 정리: "인정하는 한계 + 그럼에도 남는 핵심 우려 + 조건부 제안"

### R8: 조언가 종합 판정 (Adjudication)

- **목적:** 6개 관점에서의 종합 평가 + 보고서 통합 권고
- **에이전트:** 조언가 (Claude Opus)
- **판정 기준:**

  #### A. 교차심문 승패 평가
  | 평가 항목 | 기준 |
  |-----------|------|
  | R3 교차심문 | 반박가의 질문이 입안자의 논거를 유효하게 흔들었는가? |
  | R5 교차심문 | 입안자의 질문이 반박가의 논거를 유효하게 흔들었는가? |
  | 인정된 약점 | 양측이 스스로 인정한 한계는 무엇인가? |
  | 미해결 쟁점 | 교차심문으로도 해소되지 않은 쟁점은 무엇인가? |

  #### B. 6관점 종합 판정
  | 관점 | 판정 (찬성 우위 / 반대 우위 / 무승부) | 근거 |
  |------|---------------------------------------|------|
  | 교육부 | | |
  | 현장교사 | | |
  | 지역사회 | | |
  | 대학원생 | | |
  | 대학 내부 | | |
  | 연구방법론 | | |

  #### C. 보고서 통합 권고
  1. 5장(당위성)에 반영할 강화된 논거
  2. 6장(쟁점 대응)에 반영할 교차심문 Q&A
  3. 8장(향후 과제)에 반영할 미해결 쟁점·데이터 갭

## 실행 가이드

### 토론 개시

```
토론 주제: "[주제를 여기에]"
```

사회자(사용자)가 주제를 제시하면, 아래 순서로 에이전트를 실행한다.

### 에이전트 실행 순서

| 순서 | 에이전트 | 실행 환경 | 입력 |
|------|----------|-----------|------|
| 1 | 분석가 | Claude Opus (subagent) | 토론 주제 |
| 2 | 입안자 | Claude Opus (subagent) | R1 팩트시트 |
| 3-a | 반박가 (질문) | Codex GPT-5.4 xhigh | R1 + R2 |
| 3-b | 입안자 (답변) | Claude Opus (subagent) | R3-a 질문 |
| 4 | 반박가 | Codex GPT-5.4 xhigh | R1 + R2 + R3 |
| 5-a | 입안자 (질문) | Claude Opus (subagent) | R1 + R4 |
| 5-b | 반박가 (답변) | Codex GPT-5.4 xhigh | R5-a 질문 |
| 6 | 입안자 | Claude Opus (subagent) | R1~R5 전체 |
| 7 | 반박가 | Codex GPT-5.4 xhigh | R1~R6 전체 |
| 8 | 조언가 | Claude Opus (subagent) | R1~R7 전체 |

### 교차심문 실행 방식

교차심문(R3, R5)은 2단계로 실행한다:
1. **질문 생성:** 질문자 에이전트가 상대 입론을 읽고 교차심문 질문 3~5개 생성
2. **답변 생성:** 답변자 에이전트가 질문을 읽고 답변 (질문별로 인정/부인/조건부 인정)

### 토론 기록 저장

모든 토론 기록은 아래 경로에 저장:
```
reports/debates/NNN_[토론주제_요약].md
```

파일 내부는 `DEBATE_TEMPLATE.md` 템플릿을 따른다.

## 절대 원칙 (모든 라운드 공통)

1. **데이터 기반:** 모든 주장·질문·답변에 수치적 근거와 출처 필수
2. **전공명 원어 보존:** 임의 표준화 금지
3. **데이터 갭 명시:** 근거 없는 주장은 "데이터 미확보"로 표기
4. **보고서 활용:** 모든 출력은 집필가의 보고서 재료
5. **한국어 출력:** 정책 문서 수준 격식체
